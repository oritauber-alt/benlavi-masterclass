## לפני האקתון

### טופס בריף עסקי

ממולא על ידי המשתתף ברגע ההרשמה או עד שבוע לפני. שאלות:

1. שם העסק + תחום
2. מה את/ה עושה ביום יום? (2-3 משפטים)
3. מה הדבר שהכי גוזל לך זמן בעסק?
4. האם יש משימה שחוזרת על עצמה שהיית שמח לאוטמט?
5. מה הניסיון שלך עם כלי AI? (רמה 1-5)
6. לוגו (אם יש)
7. צבעי מותג
8. לינק לאתר קיים (אם יש - נשלוף סגנון)

### פלואו אוטומציות - מרגע התשלום עד האקתון

**שלב 1: תשלום**
- משתתף משלם בדף נחיתה
- מייל אוטומטי "ברוך הבא" + לינק לטופס בריף עסקי
- נוצרת שורה ב-Supabase (שם, מייל, טלפון, סטטוס: "שילם")
- שם משתמש + סיסמה נוצרים אוטומטית

**שלב 2: בריף ממולא**
- משתתף ממלא טופס בריף (8 שאלות)
- אוטומציה יוצרת סקיל (SKILL.md) מותאם אישית מהתשובות
- הסקיל + צבעי מותג + לוגו עולים ללוח הבקרה האישי
- סטטוס מתעדכן ל"בריף הושלם"
- מייל "הכל מוכן, נתראה ב-30/4" + שם משתמש וסיסמה

**שלב 3: יום האקתון**
- משתתף נכנס עם שם משתמש + סיסמה
- רואה הכל מוכן: סקיל, תבניות, מדריכים

**טכנולוגיה:** Supabase (DB + Auth) + Next.js (מערכת) + API ליצירת סקילים

## ארכיטקטורת המערכת

### טכנולוגיה
- **Next.js** App Router + TypeScript + Tailwind | תיקיית `app/`
- **Supabase** (DB + Auth + Storage)
- **Drizzle ORM** (type-safe queries)
- **Resend** (מיילים אוטומטיים)
- **Claude API** (יצירת סקיל מותאם)
- **טופס חיצוני** (Tally.so / Google Forms) לבריף
- **shadcn/ui** לקומפוננטים
- Hebrew RTL, Noto Sans Hebrew

### DB Schema

**participants**
| עמודה | סוג | הערות |
|-------|-----|-------|
| id | uuid PK | מקושר ל-auth.users |
| email | text unique | |
| full_name | text | |
| phone | text | |
| business_name | text | מהבריף |
| business_domain | text | מהבריף |
| temp_password | text | נשמר לשליחה חוזרת |
| status | text | paid / brief_submitted / skill_generated / ready |
| daily_work | text | בריף Q2 |
| time_consuming_task | text | בריף Q3 |
| automatable_task | text | בריף Q4 |
| ai_experience | integer | 1-5 |
| brand_colors | text | |
| logo_url | text | Supabase storage |
| website_url | text | |
| selected_site_type | text | landing / info / portfolio / catalog |
| selected_design_style | text | modern-minimal / bold-colorful / elegant-business / creative-young |
| selected_agent_track | text | content / invoice / day-manager |
| current_step | integer | 1-4 |

**skills** - participant_id (FK unique), skill_content (markdown text), generated_at
**prompts** - site_type, design_style, prompt_template עם {{skill}} placeholder, display_name_he (16 שורות seed)
**agent_tracks** - track_key, title_he, subtitle_he, description_he, prompt_template, required_mcps (3 שורות seed)
**mcp_guides** - mcp_name, title_he, guide_content_md (4 שורות seed)

### Auth
- Supabase Auth עם email + password
- יצירת משתמשים דרך API (לא self-signup)
- סיסמה: `[שם]-[4 ספרות]` (למשל: omer-3847)
- temp_password נשמר ב-DB לשליחה חוזרת
- Middleware מגן על /dashboard
- RLS: כל משתמש רואה רק את עצמו

### פלואו אוטומציות
```
תשלום (webhook) ──→ יצירת user ב-Supabase Auth
                 ──→ שורה ב-participants (status: paid)
                 ──→ מייל "ברוך הבא" + לינק לבריף

בריף (webhook) ───→ עדכון participants עם תשובות
               ───→ קריאה ל-Claude API → יצירת SKILL.md
               ───→ שמירה ב-skills table
               ───→ status → ready
               ───→ מייל "הכל מוכן" + credentials

יום האקתון ────→ login עם email + password
           ────→ dashboard עם סקיל + תבניות + מדריכים
```
